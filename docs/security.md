# Модель безопасности

Familia хранит **очень чувствительные личные данные**: здоровье членов семьи, финансы, расписания, школьные планы, супружеские вопросы. Среди читателей могут быть дети. Сломанный контроль доступа здесь раскрывает не звезды GitHub, а домашнюю жизнь.

Эта страница — публичное краткое описание. Полная модель угроз хранится в личных заметках сопровождающего: она намеренно не перечисляет каждый вектор атаки дословно, чтобы сам документ не стал подсказкой для атакующего.

Как **сообщить** об уязвимости: см. [`../SECURITY.md`](../SECURITY.md).

## Модель участников

- **Владелец / родители** — полный админский доступ, могут писать любому, видят все.
- **Другие взрослые участники** (супруги, родственники супруга, если добавлены) — свои области `private:` и `pair:`, а также темы, к которым есть связи.
- **Дети как отдельные участники** (когда ребенок вырос и получил телефон) — только своя область `private:`; по умолчанию не читают данные родителей. Асимметрия намеренная.
- **Няни / помощники вне семьи** (`role: member`, добавляются позже) — только темы, к которым им явно выданы связи.

## Гарантии

1. **Запрет по умолчанию** на каждом слое: политика, список доступа memX, достижимость в графе.
2. **Зарезервированные структурные ключи** — `shared:roles.*`, `shared:family.graph*`, `shared:topics.graph*` — запрещены на запись для **всех участников чата**, включая админов. Изменения проходят через CLI `familia` на ВМ.
3. **Проверка достижимости при записи тега**: участник не может пометить запись идентификатором, к которому сам не имеет доступа. Это защищает от враждебной подстановки.
4. **Защита от утечки словаря**: запрос LLM на каждом ходе видит только названия тем, доступных действующему участнику.
5. **Безопасное значение по умолчанию для детей-участников**: `role: child` блокирует обратный обход `parent_of`. Это жестко задано в коде, а не настраивается графом.
6. **Маршрутизация подтверждений**: подтверждения записи отправляются только во входящий чат, без рассылки.
7. **Кодек на sentinel**: сшивки профилей других участников содержат `__familia_acl_v1: true`. Парсеры отклоняют отсутствующий или неверный sentinel.
8. **Закрытый отказ при ошибке разбора**: поврежденные графы -> пустое множество достижимых объектов, запись отклоняется, чтение возвращает "no value". Нет молчаливого разрешения.
9. **Полнота аудита**: каждое привилегированное решение списка доступа пишет запись; каждое изменение графа через CLI пишет `graph_edit`.
10. **Права на журнал аудита**: `audit.jsonl` имеет `chmod 0600`.
11. **Гигиена админского ключа**: CLI читает админский ключ memX из `/etc/familia/admin.key` (режим 0400, владелец root). Запасной путь через переменную окружения есть для разработки и сопровождается громким предупреждением.

## Что не гарантируется

Система **не** защищает от:

- **Физического доступа или root-доступа к ВМ**. Владелец хоста может прочитать все. Если это важно, используйте шифрованные тома.
- **Прошлых данных после отзыва связи**. Отзыв действует только вперед: уже прочитанное нельзя сделать непрочитанным.
- **Ошибки LLM при выборе тегов внутри доступной области**. LLM может выбрать более широкий набор тегов, чем хотел пользователь; это смягчается сообщением подтверждения и аудитом, но не запрещается кодом.
- **Доверенного, но любопытного админа**. Журнал аудита записывает его действия для проверки; кроме этого злоупотребление не блокируется.
- **Согласия детей без собственного голоса**. Решения о данных несовершеннолетнего принимают опекуны, пока ребенок не станет отдельным участником.

## В области ответственности: сообщайте

- Обход списка доступа: любой способ прочитать чужую область `private:` / `pair:` без разрешающей связи.
- Обход подтверждения связи между участниками: любой способ получить доступ к другому участнику без подтверждения админом.
- Повышение прав через prompt injection: сообщение в чате, которое заставляет бота писать или читать неразрешенную область, манипулируя LLM.
- Подмена журнала аудита: любой способ для участника чата подавить или изменить запись.
- Утечка учетных данных: любой способ извлечь ключ memX или значение `.env` через бота.
- Выход из песочницы: любой способ вызовом инструмента выйти за пределы песочницы bubblewrap.

## Вне области ответственности

- Запросы функций, вопросы производительности, ошибки интерфейса.
- Самоотказ в обслуживании через законные, но дорогие запросы.
- Все, что требует вредоносного админа: админ находится внутри границы доверия.
- Уязвимости в верхнеуровневых `nanobot` или `memX`, которые недостижимы через путь выполнения familia. Сообщайте о них в соответствующие проекты.
